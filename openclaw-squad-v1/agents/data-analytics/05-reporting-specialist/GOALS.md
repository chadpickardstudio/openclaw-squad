# GOALS — Reporting Specialist

## Role-Level OKRs
The Reporting Specialist inherits squad OKRs from the Analytics Lead and owns
these reporting and visualization objectives.

---

### OKR 1: Report & Dashboard Delivery
**Objective:** Deliver all reports and dashboards on time with high quality.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR1.1 Report/dashboard delivery rate | 100 % on-time | 🔲 Not started |
| KR1.2 Dashboard uptime | ≥ 99 % | 🔲 Not started |
| KR1.3 Scheduled report adherence | 100 % delivered on schedule | 🔲 Not started |

### OKR 2: Stakeholder Satisfaction
**Objective:** Ensure all data consumers find reports clear and actionable.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR2.1 Stakeholder satisfaction score | ≥ 4/5 | 🔲 Not started |
| KR2.2 Visualization accuracy | 100 % — no misleading charts | 🔲 Not started |
| KR2.3 Report revision requests | ≤ 1 per deliverable | 🔲 Not started |

### OKR 3: Visualization Standards & Efficiency
**Objective:** Maintain consistent, efficient reporting infrastructure.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR3.1 Template reuse rate | ≥ 70 % of new reports use templates | 🔲 Not started |
| KR3.2 Dashboard load time | ≤ 3 seconds for standard dashboards | 🔲 Not started |
| KR3.3 Style guide compliance | 100 % of reports follow visual standards | 🔲 Not started |

---

## Collaboration Goals
- **Daily:** Update insights-tracker.md with report delivery status.
- **Per deliverable:** Deliver dashboard/report to @analytics-lead for quality
  gate review before stakeholder release.
- **Weekly:** Sync with @data-engineer on data feed health affecting dashboards.
- **Cross-department:** Gather feedback from dashboard consumers (Marketing,
  Finance, Sales) to improve report quality.
- **Standup participation:** Respond to @analytics-lead daily standup
  collection with current report queue and blockers.
