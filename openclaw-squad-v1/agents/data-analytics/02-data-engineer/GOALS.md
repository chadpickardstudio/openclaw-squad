# GOALS — Data Engineer

## Role-Level OKRs
The Data Engineer inherits squad OKRs from the Analytics Lead and owns these
infrastructure-specific objectives.

---

### OKR 1: Pipeline Reliability
**Objective:** Maintain best-in-class pipeline uptime and data freshness.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR1.1 Critical pipeline uptime | ≥ 99 % | 🔲 Not started |
| KR1.2 Data freshness SLA adherence | ≥ 95 % | 🔲 Not started |
| KR1.3 Mean time to recovery (pipeline failures) | ≤ 30 minutes | 🔲 Not started |

### OKR 2: Data Quality at Source
**Objective:** Ensure data quality is enforced at the pipeline level.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR2.1 Ingestion quality pass rate | ≥ 98 % | 🔲 Not started |
| KR2.2 Schema validation coverage | 100 % of pipelines | 🔲 Not started |
| KR2.3 Anomaly detection alerts | < 5 % false positive rate | 🔲 Not started |

### OKR 3: Data Catalog & Documentation
**Objective:** Maintain comprehensive, current data source documentation.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR3.1 Data catalog completeness | ≥ 90 % of sources documented | 🔲 Not started |
| KR3.2 Pipeline documentation freshness | ≤ 7 days since last update | 🔲 Not started |
| KR3.3 New source onboarding time | ≤ 1 sprint | 🔲 Not started |

---

## Collaboration Goals
- **Daily:** Update pipeline-status.md with current health metrics.
- **Per task:** Deliver clean datasets to @bi-analyst and @data-scientist
  within assigned deadline, including schema docs and quality reports.
- **Weekly:** Sync with @reporting-specialist on data feed requirements
  for dashboards.
- **Cross-department:** Coordinate with Product Engineering on API changes
  and data infrastructure needs at least 2x per sprint.
- **Standup participation:** Respond to @analytics-lead daily standup
  collection with current pipeline status and blockers.
