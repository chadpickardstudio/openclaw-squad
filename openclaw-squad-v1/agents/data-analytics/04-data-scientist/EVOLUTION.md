# EVOLUTION — Data Scientist

## Purpose
The Data Scientist evolves through structured self-reflection and methodological
improvement cycles.

---

## Self-Reflection Protocol

### Trigger Cadence
1. **Per-Sprint Review** — after every sprint
2. **Monthly Deep Reflection** — comprehensive analysis
3. **Incident-Triggered** — after model failures or statistical errors

### Per-Sprint Reflection Template
```markdown
## Sprint [N] — Data Scientist Self-Reflection

### What went well?
- [Model/experiment that produced strong business impact]

### What could improve?
- [Methodology gaps, slow iteration, compute inefficiency]

### Model/experiment delivery this sprint
- Projects delivered: [X] / [Assigned] = [%]
- Model accuracy vs threshold: [summary]
- Experiments concluded: [X]

### Methodology quality
- Statistical reviews conducted: [X]
- Errors caught pre-delivery: [X]
- Reproducibility verified: [Y/N for each project]

### Compute utilization
- Allocated: [N] | Used: [M] = [%]

### Evolution action items
- [ ] [Concrete improvement for next sprint]
```

### Monthly Additions
- **Algorithm portfolio review** — are we using the right tools for each problem?
- **Model performance audit** — are deployed models degrading?
- **Cross-department feedback** — are experiment results driving action?
