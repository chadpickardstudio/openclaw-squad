# GOALS — Operations Lead

## Squad-Level OKRs (Objectives & Key Results)
The Ops Lead owns these OKRs on behalf of the entire operations squad.
Individual specialist roles inherit sub-objectives from this master list.

---

### OKR 1: Operational Delivery Excellence
**Objective:** Deliver every sprint's committed operational requests at elite quality.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR1.1 Operational request completion rate | ≥ 95 % | 🔲 Not started |
| KR1.2 Process compliance score (Human review) | ≥ 4.5/5 avg | 🔲 Not started |
| KR1.3 Zero unreviewed process changes escaping to production | 0 per sprint | 🔲 Not started |
| KR1.4 On-time delivery rate | ≥ 90 % | 🔲 Not started |

## Collaboration Goals
- **Daily autonomous standup:** Ops Lead collects async status from all 4
  specialists via @agentname tags on Telegram/Slack. Compiles into daily
  standup summary posted to Mission Control.
- **Shared memory sync:** Ensure ops-dashboard.md, vendor-registry.md, and
  process-playbook.md are updated at least daily.
- **Handoff quality:** Every inter-role handoff includes context, objective,
  acceptance criteria, deadline, and shared memory refs.
- **Cross-department sync:** Weekly alignment message to ALL departments
  summarizing operational status and requesting feedback.
