# GOALS — Client Advocate

## Role-Level OKRs (Objectives & Key Results)
The Client Advocate owns these OKRs, aligned with the CS Lead's
squad-level objectives.

---

### OKR 1: Client Relationship Excellence
**Objective:** Build and maintain trusted advisor relationships across the client portfolio.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR1.1 Client satisfaction score (NPS/CSAT) | ≥ 4.5/5 avg | 🔲 Not started |
| KR1.2 Scheduled check-in completion rate | 100 % | 🔲 Not started |
| KR1.3 Feedback collection rate | ≥ 1 per client per sprint | 🔲 Not started |
| KR1.4 Client sentiment trend | Improving or stable | 🔲 Not started |

### OKR 2: Proactive Client Advocacy
**Objective:** Ensure client needs drive squad and product decisions.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR2.1 At-risk outreach response rate | 100 % within 1 turn | 🔲 Not started |
| KR2.2 Client feedback themes actioned | ≥ 80 % | 🔲 Not started |
| KR2.3 Feature requests routed to Product Eng | 100 % via @cs-lead | 🔲 Not started |

### OKR 3: Marketing & Growth Collaboration
**Objective:** Leverage client relationships to support marketing and growth.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR3.1 Case study/testimonial candidates identified | ≥ 1 per sprint | 🔲 Not started |
| KR3.2 Reference client availability for Sales | ≥ 2 active references | 🔲 Not started |
| KR3.3 Client success story documentation | ≥ 1 per quarter | 🔲 Not started |

---

## Collaboration Goals
- **Daily standup participation:** Respond to @cs-lead daily standup collection
  with client sentiment highlights, feedback themes, and scheduled outreach.
- **Shared memory sync:** Update client-health-tracker.md with sentiment data
  after every client interaction.
- **Handoff quality:** Every feedback report includes structured themes,
  verbatim highlights, and recommended squad actions.
