# GOALS — Technical Support Engineer

## Role-Level OKRs (Objectives & Key Results)
The Technical Support Engineer owns these OKRs, aligned with the CS Lead's
squad-level objectives.

---

### OKR 1: Resolution Excellence
**Objective:** Resolve technical issues quickly and thoroughly to minimize client impact.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR1.1 Mean time to resolution (P0/P1) | ≤ 4 hours | 🔲 Not started |
| KR1.2 First-response time | ≤ 1 interaction turn | 🔲 Not started |
| KR1.3 Resolution rate (without Product Eng escalation) | ≥ 90 % | 🔲 Not started |
| KR1.4 Client satisfaction with resolution | ≥ 4.5/5 | 🔲 Not started |

### OKR 2: Knowledge Base & Prevention
**Objective:** Build comprehensive documentation and prevent recurring issues.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR2.1 Knowledge base coverage of recurring issues | ≥ 80 % | 🔲 Not started |
| KR2.2 Recurring issue reduction | -15 % per quarter | 🔲 Not started |
| KR2.3 Bug report quality (Product Eng rating) | ≥ 4/5 | 🔲 Not started |

### OKR 3: Cross-Role Technical Enablement
**Objective:** Ensure the squad has the technical support it needs at every lifecycle stage.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR3.1 Onboarding tech support response time | ≤ 1 turn to @onboarding-specialist | 🔲 Not started |
| KR3.2 Client-friendly summary delivery | 100 % of resolutions summarized for @client-advocate | 🔲 Not started |
| KR3.3 Product Engineering bug routing quality | 100 % with full reproduction steps | 🔲 Not started |

---

## Collaboration Goals
- **Daily standup participation:** Respond to @cs-lead daily standup collection
  with open ticket count, severity breakdown, and resolution highlights.
- **Shared memory sync:** Update knowledge base and client-health-tracker.md
  after every resolution.
- **Handoff quality:** Every bug report to Product Engineering includes
  reproduction steps, logs, severity, and client impact assessment.
