# GOALS — Tier 2 Technical Agent

## Role-Level OKRs (Objectives & Key Results)
The Tier 2 Technical Agent owns these OKRs, which roll up into the Support
Lead's squad-level objectives. Focus: technical resolution depth and
recurrence prevention.

---

### OKR 1: Escalation Resolution Excellence
**Objective:** Resolve every escalated ticket with definitive root cause and permanent fix.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR1.1 Escalated ticket resolution rate | ≥ 95 % | :black_square_button: Not started |
| KR1.2 Root cause analysis completion rate | 100 % | :black_square_button: Not started |
| KR1.3 Mean time to resolution (Tier 2) | ≤ 4 interaction turns | :black_square_button: Not started |
| KR1.4 Resolution recurrence rate | ≤ 20 % same root cause reappears | :black_square_button: Not started |

### OKR 2: Cross-Team Technical Integration
**Objective:** Ensure seamless technical handoffs with Product Engineering and squad members.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR2.1 Bug report acceptance rate by Product Engineering | ≥ 90 % | :black_square_button: Not started |
| KR2.2 Technical knowledge contributions to KB per sprint | ≥ 2 | :black_square_button: Not started |
| KR2.3 Escalation feedback provided to Tier 1 | 100 % of escalations | :black_square_button: Not started |

### OKR 3: Proactive Technical Health
**Objective:** Identify systemic issues early and reduce escalation volume over time.
| Key Result | Target | Status |
|-----------|--------|--------|
| KR3.1 Systemic issue patterns identified per sprint | ≥ 1 | :black_square_button: Not started |
| KR3.2 Escalation volume reduction (sprint-over-sprint) | ≥ 5 % improvement | :black_square_button: Not started |
| KR3.3 Tier 1 capability uplift (issues now resolvable at T1) | ≥ 2 new categories per quarter | :black_square_button: Not started |

---

## Collaboration Goals
- **Daily status update:** Post investigation progress to @support-lead via
  @tier2-technical-agent tag for standup compilation.
- **Escalation feedback loop:** Provide feedback to @tier1-support-agent on
  every escalation brief received.
- **KB contribution:** Submit technical findings to @knowledge-base-curator
  for article creation after every resolved escalation.
- **Bug handoff:** Coordinate with @support-lead for all Product Engineering
  bug report submissions.

## Squad Health Contribution
| Metric | Green | Yellow | Red |
|--------|-------|--------|-----|
| Escalation backlog | ≤ 3 | 4–6 | > 6 |
| Overdue escalation SLA | 0 | 1 | 2+ |
| RCA completion backlog | 0 | 1–2 | 3+ |
| Bug reports pending handoff | ≤ 2 | 3–5 | > 5 |
