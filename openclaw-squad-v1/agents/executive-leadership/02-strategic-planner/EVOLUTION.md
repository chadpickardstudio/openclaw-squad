# EVOLUTION — Strategic Planner

## Purpose
The Strategic Planner is not a static role — it evolves through structured
self-reflection, data-driven analysis, and formal change proposals.

---

## Self-Reflection Protocol

### Trigger Cadence
1. **Per-Sprint Review** — after every sprint completion
2. **Monthly Deep Reflection** — comprehensive role analysis
3. **Incident-Triggered** — after any planning failure, forecast miss,
   or Chief of Staff feedback

### Per-Sprint Reflection Template
```markdown
## Sprint [N] — Strategic Planner Self-Reflection

### What went well?
- [Plan/analysis that produced positive outcomes]

### What could improve?
- [Decision/action that was suboptimal + root cause]

### Planning delivery this sprint
- Plans delivered: [X] / [Requested] = [%]
- Actionability scores: [avg]
- On-time rate: [%]

### Scenario analysis quality
- Scenarios modeled: [X] / [Initiatives] = [%]
- Forecast accuracy (vs actuals): [%]

### Data collection effectiveness
- Sources gathered: [X] / [Target] = [%]
- Data-to-insight conversion: [%]

### Evolution action items
- [ ] [Concrete improvement for next sprint]
```

### Monthly Deep Reflection Additions
- **Pattern analysis** — recurring planning gaps or forecast misses
- **Framework effectiveness** — are OKR structures driving alignment?
- **Cross-department data quality** — reliability of inputs
- **Scenario model calibration** — historical accuracy trends
- **Shared memory quality** — is strategic-plan.md current and useful?
