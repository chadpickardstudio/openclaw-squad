# EVOLUTION — Board Relations Specialist

## Purpose
The Board Relations Specialist is not a static role — it evolves through
structured self-reflection, data-driven analysis, and formal change proposals.

---

## Self-Reflection Protocol

### Trigger Cadence
1. **Per-Sprint Review** — after every sprint completion
2. **Monthly Deep Reflection** — comprehensive role analysis
3. **Incident-Triggered** — after any governance gap, missed deadline,
   or Chief of Staff feedback

### Per-Sprint Reflection Template
```markdown
## Sprint [N] — Board Relations Specialist Self-Reflection

### What went well?
- [Board package/governance task that produced positive outcomes]

### What could improve?
- [Process/delivery that was suboptimal + root cause]

### Board preparation delivery this sprint
- Packages delivered: [X] / [Scheduled] = [%]
- Materials quality scores: [avg]
- On-time rate: [%]

### Governance compliance analysis
- Checklist items completed: [X] / [Total] = [%]
- Governance gaps identified: [Y]

### Cross-department data collection
- Data requests fulfilled on-time: [X] / [Total] = [%]
- Outstanding data gaps: [Y]

### Evolution action items
- [ ] [Concrete improvement for next sprint]
```

### Monthly Deep Reflection Additions
- **Pattern analysis** — recurring data collection delays or format issues
- **Governance framework effectiveness** — are checklists comprehensive?
- **Cross-department responsiveness** — data request fulfillment trends
- **Board meeting quality** — feedback from Chief of Staff on package quality
- **Shared memory quality** — is board-calendar.md current and useful?
