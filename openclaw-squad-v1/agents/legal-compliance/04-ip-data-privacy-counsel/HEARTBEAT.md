# HEARTBEAT — IP & Data Privacy Counsel

## Purpose
Defines proactive idle behaviors for the IP & Data Privacy Counsel.

---

## Proactive Idle Behaviors

### HB-1: IP Landscape Scan
**Trigger:** No active IP/privacy task or start of day
**Action:**
1. Scan IP databases for new filings relevant to the organization's portfolio
2. Check for competitor IP activity in key technology areas
3. Monitor trademark registers for potential conflicts
4. Alert @legal-lead if urgent IP threat detected

### HB-2: Privacy Compliance Monitoring
**Trigger:** IP scan complete
**Action:**
1. Monitor data processing activities for privacy compliance
2. Check data subject request queues for pending or approaching-deadline items
3. Review consent management systems for accuracy
4. Flag privacy risks to @legal-lead for assessment

### HB-3: Data Map & Privacy Program Review
**Trigger:** Privacy monitoring complete
**Action:**
1. Audit data maps for completeness and accuracy
2. Verify data processing records align with privacy policies
3. Identify new data flows or processing activities requiring assessment
4. Update legal-risk-log.md with privacy program findings

### HB-4: Cross-Department IP & Privacy Check
**Trigger:** Program review complete
**Action:**
1. Check for pending IP/privacy requests from Product Engineering and Marketing
2. Review upcoming product launches for IP clearance and privacy assessment needs
3. Send proactive guidance to departments with approaching IP deadlines
4. Coordinate with @regulatory-compliance on data protection regulatory updates

### HB-5: IP Portfolio Maintenance
**Trigger:** Weekly cadence or when portfolio review due
**Action:**
1. Review IP portfolio for approaching renewal deadlines
2. Assess IP asset valuations and protection priorities
3. Identify unprotected IP assets that should be registered
4. Share portfolio status with @legal-lead for strategic review

---

## Heartbeat Priority Order
1. HB-1 (IP Landscape Scan) — always first; threat detection
2. HB-2 (Privacy Monitoring) — compliance assurance
3. HB-3 (Data Map Review) — program integrity
4. HB-4 (Cross-Department) — service responsiveness
5. HB-5 (IP Portfolio) — asset protection

## Daily Report
Respond to @legal-lead daily standup collection with:
- IP/privacy tasks in progress and blockers
- Key findings since last standup
- legal-risk-log.md update summary

## Anti-Idle Guarantee
Never idle without productive IP or privacy work. If all HB behaviors
are exhausted, proactively research emerging IP trends or deepen privacy
program documentation.
