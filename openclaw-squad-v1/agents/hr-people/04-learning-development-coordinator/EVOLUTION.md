# EVOLUTION — Learning & Development Coordinator

## Purpose
The Learning & Development Coordinator is not a static role — it evolves
through structured self-reflection, data-driven analysis, and formal
change proposals.

---

## Self-Reflection Protocol

### Trigger Cadence
1. **Per-Sprint Review** — after every sprint completion
2. **Monthly Deep Reflection** — comprehensive role analysis
3. **Incident-Triggered** — after any program failure, compliance gap,
   or HR Lead feedback

### Per-Sprint Reflection Template
```markdown
## Sprint [N] — L&D Coordinator Self-Reflection

### What went well?
- [Program/initiative that produced positive outcomes]

### What could improve?
- [Action that was suboptimal + root cause]

### Training delivery this sprint
- Programs delivered: [X] / [Planned] = [%]
- Avg completion rate: [%]
- Avg assessment score: [X/5]

### Skill gap analysis
- Assessments completed: [X]
- Critical gaps identified: [X]
- Gaps addressed via training: [X]

### Learning ROI
- Programs with measured business impact: [X] / [Total]
- Avg Learning NPS: [Score]

### Evolution action items
- [ ] [Concrete improvement for next sprint]
```

### Monthly Deep Reflection Additions
- **Pattern analysis** — recurring skill gaps across departments
- **Program effectiveness trends** — completion and impact over time
- **Learning modality analysis** — which formats drive best outcomes
- **Career path utilization** — are employees using career frameworks?
