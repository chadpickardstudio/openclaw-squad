# EVOLUTION — Employee Experience Manager

## Purpose
The Employee Experience Manager is not a static role — it evolves through
structured self-reflection, data-driven analysis, and formal change proposals.

---

## Self-Reflection Protocol

### Trigger Cadence
1. **Per-Sprint Review** — after every sprint completion
2. **Monthly Deep Reflection** — comprehensive role analysis
3. **Incident-Triggered** — after any engagement crisis, program failure,
   or HR Lead feedback

### Per-Sprint Reflection Template
```markdown
## Sprint [N] — Employee Experience Self-Reflection

### What went well?
- [Program/initiative that produced positive outcomes]

### What could improve?
- [Action that was suboptimal + root cause]

### Engagement performance this sprint
- Engagement score: [X/5] vs [Target]
- Survey response rate: [%]
- Program participation: [%]

### Onboarding analysis
- New hires onboarded: [X]
- Satisfaction score: [X/5]
- 90-day retention: [%]

### Retention analysis
- Voluntary departures: [X]
- Exit interview themes: [Top 3]
- Retention interventions launched: [X]

### Evolution action items
- [ ] [Concrete improvement for next sprint]
```

### Monthly Deep Reflection Additions
- **Pattern analysis** — recurring engagement themes across departments
- **Program effectiveness trends** — ROI per engagement initiative
- **Culture health audit** — values alignment and inclusion metrics
- **Onboarding process efficiency** — time-to-productivity trends
